259 /* ------------------------------------------------------------------------- */
260 //! FUNCTION: LTC6945_set_frf
261 //! Calculates the integer (N), fractional (NUM) and output divider (OD) SPI values
262 //! using self created 64bit math functions.
263 //!
264 //! Datasheet equations:
265 //! - fvco = fpfd*(N + F)
266 //! - frf = fvco/O
267 //! - fpfd = fref/R
268 //!
269 //! can be modified to the following equations:
270 //! - N = (int) (fvco/fpfd) = (int) frf*O*R/fref
271 //!
272 //! where:
273 //! - N = ndiv, O= odiv in the code below
274 //!
275 //! Linduino One (Arduino Uno) is limited to 32 bit floats/double/long.
276 //! 32 bit math functions will create rounding errors with the above equations,
277 //! that can result in frequency errors.
278 //! Therefore, the following code uses self created 64bit functions for 64bit integer math.
279 //!
280 //! - frf (33 bits) LTC6945 max frf/fvco = 6.0GHZ, which is 23 bit number (2^33 = 8.59G)
281 //! - fref (23 bits) LTC6945 min fref = 10MHz, which is a 23 bit number (2^23 = 8.3M)
282 //! - O (3 bits)
283 //! - R (10 bits)
284 //!
285 //! step 1: create 64 bit frf and fref numbers
286 //!
287 //! step 2: calculate O (output divider)
288 //!
289 //! step 3: get current R-divider setting
290 //!
291 //! step 4: calculate frf*O*R
292 //! - max bit count/resolution: 33b+3b+10b= 46b
293 //!
294 //! step 5: calculate N(16b), using value from step 1
295 //! - N = (int) frf*O*R/fref
296 //! - max bit count/resolution: 46b-23b = 13b
297 //!
298 //! @return void
299 void LTC6945_set_frf();
