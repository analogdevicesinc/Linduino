271 /* ------------------------------------------------------------------------- */
272 //! FUNCTION: LTC6947_set_frf
273 //! Calculates the integer (N), fractional (NUM) and output divider (OD) SPI values
274 //! using self created 64bit math functions.
275 //!
276 //! Datasheet equations:
277 //! - fvco = fpfd*(N + F)
278 //! - frf = fvco/O
279 //! - fpfd = fref/R
280 //! - F=NUM/2^18 = NUM/262144
281 //!
282 //! can be modified to the following equations:
283 //! - N = (int) (fvco/fpfd) = (int) frf*O*R/fref
284 //! - NUM = (int) [262144 * (frf*O*R/fref -N)]
285 //!
286 //! where:
287 //! - N = ndiv, O= odiv, NUM=fracN_num in the code below
288 //!
289 //! Linduino One (Arduino Uno) is limited to 32 bit floats/double/long.
290 //! 32 bit math functions will create rounding errors with the above equations,
291 //! that can result in frequency errors.
292 //! Therefore, the following code uses self created 64bit functions for 64bit integer math.
293 //!
294 //! - frf (33 bits) LTC6947 max frf/fvco = 6GHZ, which is 33 bit number (2^33 = 8.59G)
295 //! - fref (23 bits) LTC6947 min fref = 10MHz, which is a 23 bit number (2^23 = 8.3M)
296 //! - O (3 bits)
297 //! - R (5 bits)
298 //!
299 //! step 1: create 64 bit frf and fref numbers
300 //!
301 //! step 2: calculate O (output divider)
302 //!
303 //! step 3: get current R-divider setting
304 //!
305 //! step 4: calculate frf*O*R
306 //! max bit count/resolution: 33b+3b+5b= 41b
307 //!
308 //! step 5: calculate N(10b), using value from step 1
309 //! N = (int) frf*O*R/fref
310 //! max bit count/resolution: 41b-23b = 18b
311 //!
312 //! step 6: calculate NUM (18b)
313 //! - NUM = (int) [262144 * (frf*O*R/fref -N)]
314 //! - NUM = (int) [2^18 * (frf*O*R/fref -N)]
315 //!
316 //! to get the correct value the above calculation are performed like this
317 //! - NUM =((int)((frf*O*R/fref)<<23) - N<<23 + rounding) >> 5, where << & >> are bit shift left, right
318 //! - note 1: << 23b was chosen, because step one use 41b, 41b +23b = 64b (max number for 64 bit math)
319 //! - note 2: >> 5 was chosen because 23-5 = 2^18, which is the same as multiplying by 262144
320 //! - note 3: the rounding variable is equivalent to adding 0.5 to the NUM number. the integer operation always
321 //! rounds down, so adding a 0.5 to the number before rounding down ensures the closest value is selected.
322 //! In this function the lower 5 bits of (frf*O*R/fref)<<23) - N<<23 (possible variable 0-31)
323 //! are check to see if they are >=16.
324 //! If they are then 16 is added to this number before performing the int and >>5 functions.
325 //! @return void
326 void LTC6947_set_frf();
